# neptun4hass

[![HACS Compatible](https://img.shields.io/badge/HACS-Custom-orange.svg)](https://github.com/hacs/integration)
[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)

Кастомная интеграция [Home Assistant](https://www.home-assistant.io/) для системы защиты от протечек **Neptun ProW+ WiFi** производства [SST](https://sst.ru/).

Полное управление устройством через локальный TCP-протокол, без облака: мониторинг проводных и беспроводных датчиков, счётчики воды, управление кранами и режимом уборки.

## Возможности

- Управление кранами (открытие / закрытие)
- Режим уборки — временно отключает реакцию на датчики протечки
- Мониторинг проводных датчиков протечки (до 4 линий)
- Мониторинг беспроводных датчиков протечки
- Счётчики воды (м³) для линий, настроенных как счётчики
- Уровень сигнала и заряд батареи беспроводных датчиков
- Общая тревога (alarm)
- Диагностика: основная батарея, батареи датчиков, офлайн-датчики
- Поддержка нескольких устройств
- Настройка через UI (Config Flow), без YAML
- Русский и английский языки

## Совместимость

| Устройство | Статус |
|---|---|
| Neptun ProW+ WiFi | Полная поддержка, протестировано на реальном устройстве |
| Neptun ProW WiFi (без +) | Должно работать (тот же протокол) |

**Требования:** Home Assistant 2024.1+. Устройство и HA в одной локальной сети.

## Установка

### HACS (рекомендуется)

1. Откройте HACS в Home Assistant
2. Нажмите **...** (три точки) → **Пользовательские репозитории**
3. Добавьте `https://github.com/Kirorus/neptun4hass`, тип: **Интеграция**
4. Найдите **neptun4hass** в каталоге HACS и установите
5. Перезапустите Home Assistant
6. **Настройки → Устройства и службы → Добавить интеграцию → neptun4hass**

### Ручная установка

1. Скопируйте `custom_components/neptun4hass/` в `<config>/custom_components/` вашего HA
2. Перезапустите Home Assistant
3. **Настройки → Устройства и службы → Добавить интеграцию → neptun4hass**

## Настройка

| Поле | Обязательное | Описание |
|---|---|---|
| IP-адрес | Да | IP-адрес устройства Neptun ProW+ WiFi в локальной сети |
| Имя | Да | Имя устройства в Home Assistant. Используется как префикс в названиях сущностей |

Интеграция проверит подключение, получит MAC-адрес (уникальный идентификатор) и создаст все сущности автоматически. Для нескольких устройств — добавьте интеграцию повторно с другим IP.

`entity_id` формируется с префиксом имени из настройки, например:
- `switch.neptune_prow_valve`
- `switch.neptune_prow_cleaning_mode`

## Сущности

### Переключатели (switch)

| Сущность | Описание |
|---|---|
| **Valve** | Управление кранами. ON = открыты, OFF = закрыты |
| **Cleaning mode** | Режим уборки. Система игнорирует срабатывания датчиков |

### Бинарные сенсоры (binary_sensor)

| Сущность | Device class | Описание |
|---|---|---|
| **Alarm** | `safety` | Общая тревога. ON = протечка обнаружена |
| **Wired sensor 1..4** | `moisture` | Проводные датчики. Имена из настроек устройства |
| **Wireless sensor 1..N** | `moisture` | Беспроводные датчики. Количество определяется автоматически |

### Сенсоры (sensor)

| Сущность | Device class | Описание |
|---|---|---|
| **Counter 1..4** | `water` | Счётчики воды (м³), `state_class: total_increasing`. Только для линий, настроенных как счётчики |
| **Signal (wireless)** | — | Уровень сигнала беспроводного датчика (%). Диагностика |
| **Battery (wireless)** | `battery` | Заряд батареи беспроводного датчика (%). Диагностика |
| **Status** | — | Статус устройства: OK / Alarm / Main battery low / Sensor battery low / Sensor offline |

## Примеры автоматизаций

### Уведомление о протечке

```yaml
automation:
  - alias: "Протечка"
    trigger:
      - platform: state
        entity_id: binary_sensor.neptune_prow_alarm
        to: "on"
    action:
      - service: notify.mobile_app
        data:
          title: "Протечка!"
          message: "Обнаружена протечка. Краны закрыты."
```

### Автоотключение режима уборки через 2 часа

```yaml
automation:
  - alias: "Таймер уборки"
    trigger:
      - platform: state
        entity_id: switch.neptune_prow_cleaning_mode
        to: "on"
        for: "02:00:00"
    action:
      - service: switch.turn_off
        entity_id: switch.neptune_prow_cleaning_mode
```

### Низкий заряд батареи датчика

```yaml
automation:
  - alias: "Батарея датчика"
    trigger:
      - platform: numeric_state
        entity_id: sensor.neptune_prow_wl_hallway_battery
        below: 20
    action:
      - service: notify.mobile_app
        data:
          title: "Батарея"
          message: "Заряд беспроводного датчика ниже 20%."
```

## Тестирование без устройства

В репозитории есть мок-сервер, полностью эмулирующий протокол Neptun ProW+ WiFi:

```bash
# Терминал 1
python3 test_server.py

# Терминал 2
python3 test_client.py 127.0.0.1 6350

# Против реального устройства
python3 test_client.py <IP> 6350
```

## Устранение неполадок

| Проблема | Решение |
|---|---|
| Не находит устройство | Проверьте `nc -zv <IP> 6350`. Отключите AP Isolation на роутере. Перезагрузите Neptun |
| Пустые имена датчиков | Задайте имена в мобильном приложении Neptun |
| Счётчики воды = 0 | Линия должна быть настроена как «счётчик» в устройстве |
| "Empty response" в логах | Другой клиент (приложение, neptun2mqtt) занимает соединение. Используйте один клиент |

## Релизы для HACS

HACS показывает обновления по новым версиям из GitHub Release/тегов.

1. Обновите версию в `custom_components/neptun4hass/manifest.json` (например, `1.0.2`)
2. Сделайте commit и push в `main`
3. Создайте аннотированный тег той же версии с префиксом `v` и коротким описанием релиза:
   - `git tag -a v1.0.2 -m "Короткое описание релиза"`
   - `git push origin v1.0.2`
4. Workflow `.github/workflows/release.yml` автоматически создаст GitHub Release

Важно: версия в теге и `manifest.json` должна совпадать (`v1.0.2` -> `1.0.2`), иначе релиз не будет создан.
Если тег без описания, workflow сформирует краткие notes автоматически по коммитам.
